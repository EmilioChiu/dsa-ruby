# Recursos DSA en Ruby

Recursos curados de alta confianza para el estudio de Estructuras de Datos y Algoritmos en Ruby, orientado a entrevistas tecnicas.

## Conocimiento

### Visualizacion y Animacion
- [VisuAlgo](https://visualgo.net/en/list)
  Visualizacion interactiva de estructuras de datos (listas, stacks, queues, grafos, arboles). Ideal para entender el comportamiento dinamico de algoritmos paso a paso. Usar antes de implementar.
- [Big-O Cheat Sheet](https://www.bigocheatsheet.com)
  Referencia rapida de complejidades temporales y espaciales para estructuras de datos y algoritmos de ordenamiento. Imprimible.

### Plataformas de Practica
- [LeetCode](https://leetcode.com)
  Repositorio canonico de problemas de entrevistas. Cada problema incluye descripcion, restricciones, ejemplos y casos de prueba.
- [NeetCode 150](https://neetcode.io)
  Curaduria de 150 problemas esenciales de LeetCode organizados por patron. Roadmap principal para preparacion de entrevistas.
- [AlgoMonster](https://algo.monster)
  Explicaciones de patrones algoritmicos con enfasis en reconocimiento de problemas y tecnicas de resolucion.

### Referencia de Ruby
- [Ruby-Doc Array](https://ruby-doc.org/core/Array.html)
  Documentacion oficial de Array en Ruby. Cubre metodos nativos utiles para DSA: `push`, `pop`, `shift`, `unshift`, `slice`, `combination`, `permutation`, etc.
- [Ruby-Doc Enumerable](https://ruby-doc.org/core/Enumerable.html)
  Modulo Enumerable: `map`, `select`, `reduce`, `any?`, `all?`, `sort_by`. Esencial para escribir codigo idiomatico en Ruby.

### Libros de Texto
- _Cracking the Coding Interview_ — Gayle Laakmann McDowell (6a ed.)
  189 problemas con explicaciones detalladas. Referencia estandar para preparacion de entrevistas.
- _Introduction to Algorithms_ — Cormen, Leiserson, Rivest, Stein (CLRS, 3a ed.)
  Texto fundamental de teoria algoritmica. Usar como referencia profunda cuando el entendimiento superficial no basta.

### Curso Estructurado
- [The Odin Project — Ruby Course](https://www.theodinproject.com/paths/full-stack-ruby-on-rails/courses/ruby)
  Seccion de algoritmos y estructuras de datos en Ruby. Buen complemento para ver implementaciones en el lenguaje.

### Monotonic Stack (Patrón Específico)
- [AlgoMonster: Monotonic Stack Intro](https://algo.monster/problems/monotonic_stack_intro)
  Guía visual y teórica sobre el patrón. Excelente para entender cuándo aplicarlo.
- [NeetCode: Daily Temperatures Video](https://www.youtube.com/watch?v=cTBiBSnjO3c)
  Explicación visual de la intuición de la "sala de espera".
- [Back To Back SWE: Monotonic Stack](https://www.youtube.com/watch?v=Dq_fIltvInw)
  Explicación profunda de la teoría y por qué la complejidad es O(n).
- [LeetCode 496, 503, 901]
  Problemas recomendados para practicar este patrón específico.

## Sabiduria (Comunidades)

- [r/leetcode](https://reddit.com/r/leetcode)
  Discusion de estrategias para entrevistas, soluciones de problemas, experiencias de entrevistas. Altosenal.
- [r/ruby](https://reddit.com/r/ruby)
  Comunidad general de Ruby. Util para preguntas idiomaticas del lenguaje.
- [Stack Overflow — Ruby](https://stackoverflow.com/questions/tagged/ruby)
  Preguntas y respuestas tecnicas sobre Ruby. Buscar aqui antes de preguntar.

## Brechas

- No hay un recurso consolidado que mapee cada patron de DSA a su implementacion idiomatica en Ruby especificamente para entrevistas. La mayoria de recursos usan Python o Java. Este vacio se cubrira mediante las lecciones y learning records de este workspace.
