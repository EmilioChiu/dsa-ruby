BS en 2D + Framework de Decision w5d4

Como aplicar Binary Search cuando el input no es un array plano — y como decidir cual de los 4 patrones de BS usar en cada problema.

1. Search a 2D Matrix — El Problema

LeetCode 74: Te dan una matriz m x n con dos propiedades:

  • Cada fila esta ordenada de menor a mayor
  • El primer elemento de la fila i+1 es mayor que el ultimo de la fila i
Esto significa: si aplanas la matriz en un solo array de m * n elementos, el resultado es un array ordenado. Y BS funciona en arrays ordenados.

Ejemplo:

matrix = [
  [1, 3, 5, 7],
  [10, 11, 16, 20],
  [23, 30, 34, 60]
]
target = 3  # true
target = 13 # false
Visual: la matriz aplanda es un array ordenado
1357 | 10111620 | 23303460

fila 0 → fila 1 → fila 2 → todo ordenado

2. La Idea Clave: Mapeo 2D ↔ 1D

En BS clasico, el indice mid apunta a una posicion en el array. En una matriz, necesitamos convertir ese indice 1D a coordenadas 2D.

Formula: mid en el array "aplanado" de tamaño m * n se traduce a:

row = mid / n (cociente)
col = mid % n (resto)

Ejemplo con matriz 3 × 4:

r0 1357 indices 0..3
r1 10111620 indices 4..7
r2 23303460 indices 8..11

Si mid = 6: row = 6 / 4 = 1, col = 6 % 4 = 2 → matrix[1][2] = 16

Si mid = 9: row = 9 / 4 = 2, col = 9 % 4 = 1 → matrix[2][1] = 30

En Ruby:

def search_matrix(matrix, target)
  m = matrix.length
  n = matrix[0].length
  left = 0
  right = m * n - 1

  while left <= right
    mid = left + (right - left) / 2
    row = mid / n
    col = mid % n
    val = matrix[row][col]

    if val == target
      return true
    elsif val < target
      left = mid + 1
    else
      right = mid - 1
    end
  end

  false
end
Nota: Es exactamente el mismo BS clasico que usas para nums = [1,3,5,7,10,11,16,20,23,30,34,60]. Lo unico nuevo es el mapeo de indices.

4. Framework: Los 4 Patrones de BS

Hasta ahora has visto 4 formas distintas de aplicar BS. La clave es reconocer cual usar segun el problema.

# Patron Input Pregunta mid se descarta? Ejemplos
1 Clasico Array ordenado nums[mid] == target? Si (ya se comparo) LC 704, 35, 278
2 En respuesta Rango de valores feasible?(mid)? No (podria ser el minimo factible) LC 875, 1011, 69
3 Rotado Array rotado ordenado nums[mid] < nums[0]? Depende (buscar target o minimo) LC 33, 153, 162
4 2D Matriz m × n ordenada matrix[mid/n][mid%n] == target? Si (igual al clasico) LC 74

5. Arbol de Decision: Que patron usar?

? El input esta ordenado o se puede ordenar? Si → considera BS
1 Es un array plano ordenado? Si → BS clasico
2 No hay array, pero la respuesta esta en un rango y feasible?(x) es monotona? Si → BS en respuesta
3 Es un array ordenado pero rotado por un pivot? Si → BS rotado
4 Es una matriz 2D donde filas y columnas estan ordenadas? Si → BS 2D (mapeo)
X No calza en ningun patron de BS? → Busca otro patron (hash, pointers, etc.)
Cuidado: Que un problema pueda resolverse con BS no significa que deba. Ej: Find Peak Element (LC 162) se resuelve con BS aunque el array no este ordenado — porque la pendiente garantiza monotonicidad local.

La pregunta correcta no es "esta ordenado?" sino "existe una propiedad monotona que divida el espacio en dos?"

6. Variante: Buscar en una Matriz donde Filas y Columnas Estan Ordenadas

Existe una variante llamada Search a 2D Matrix II (LC 240) donde cada fila y cada columna estan ordenadas individualmente, pero NO hay relacion entre el final de una fila y el inicio de la siguiente:

# Cada fila esta ordenada, cada columna esta ordenada
# pero matrix[0][last] PUEDE ser > matrix[1][0]
matrix = [
  [1,  4,  7, 11],
  [2,  5,  8, 12],
  [3,  6,  9, 16]
]

Para esta variante, el mapeo 1D no funciona (no es un array plano ordenado). Se usa una tecnica de eliminacion por esquina: empezar en la esquina superior derecha y moverse izquierda/abajo.

Moraleja: No toda matriz con datos ordenados es candidata a BS 2D. Solo cuando la propiedad global de "todo esta ordenado" se cumple.

7. Ponte a Prueba

Pregunta 1

Matriz 3 × 5. Cual es el indice 1D de matrix[2][3]?

Ver respuesta
2 * 5 + 3 = 10 + 3 = 13

Pregunta 2

En la misma matriz 3 × 5, si mid = 7, a que coordenadas 2D corresponde?

Ver respuesta
row = 7 / 5 = 1, col = 7 % 5 = 2 → matrix[1][2]

Pregunta 3

Tienes un array no ordenado y no rotado. Puedes usar BS?

Ver respuesta
No directamente. BS requiere una propiedad monotona para descartar mitades. Si el array no esta ordenado ni tiene una propiedad como la pendiente de Find Peak, no puedes aplicar BS. Toca buscar con otro patron (hash, dos punteros, etc.).

Pregunta 4

Que patron usarias para encontrar la raiz cuadrada de un numero entero?

Ver respuesta
BS en respuesta (patron 2). No tienes un array, pero sabes que la respuesta esta entre 0 y x, y mid * mid <= x es una funcion monotona. Este es LC 69.

Pregunta 5

Que patron usarias para una matriz 4 × 4 donde cada fila esta ordenada pero matrix[0][3] puede ser mayor o menor que matrix[1][0]?

Ver respuesta
Ninguno de los 4 patrones de BS. Esta matriz no esta "fully sorted" (no puedes aplanarla). Necesitas la tecnica de esquina (LC 240) que no es BS puro, sino eliminacion en O(m + n).

8. Resumen

BS en 2D: cuando la matriz es virtualmente un array ordenado aplanado. Usa row = mid / cols y col = mid % cols. El resto es BS clasico.
Framework de decision: preguntate si existe una propiedad monotona que te permita descartar mitades. Si si, BS aplica. Si no, busca otro patron.
