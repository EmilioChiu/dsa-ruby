Lección 0003: Queues (Colas) y FIFO

Bienvenidos al mundo de las Queues. Si el Stack era un tubo de papas Pringles (LIFO), la Queue es la fila del supermercado: First In, First Out (FIFO).

1. El Concepto FIFO

En una Queue, el primer elemento en entrar es el primero en salir. Las operaciones principales son:

  • Enqueue (Ruby: push o <<): Añadir al final.
  • Dequeue (Ruby: shift): Quitar del principio.
  • Peek (Ruby: first): Ver el elemento frontal sin quitarlo.

Nota sobre Ruby: Usar Array#shift en Ruby tiene una complejidad de O(n) porque todos los elementos restantes deben moverse una posición a la izquierda. En una implementación ideal de Queue, dequeue debería ser O(1).

2. Implementación Ideal

Para lograr O(1) en ambas puntas, se suele usar una Doubly Linked List o un Circular Buffer. Ruby no tiene una clase Queue eficiente para algoritmos (la que trae es para multithreading), por lo que en entrevistas a menudo usamos Array o implementamos una simple.

3. ¿Cuándo usar una Queue?

  • BFS (Breadth-First Search): Exploración por niveles en árboles y grafos.
  • Buffer/Procesamiento de tareas: Procesar elementos en el orden exacto en que llegaron.
  • Sistemas de mensajería: Colas de trabajos (Sidekiq, RabbitMQ).
