0009: El Patron de 4 Punteros

Doubly Linked Lists con Nodos Centinela — la base del LRU Cache

Lección Semana 4 — Deuda Técnica 🔗 Reference: LRU Cache

1 El Problema que Resuelven los Centinelas

En una Doubly Linked List sin centinelas, cada operación necesita preguntar: "¿es esto nil?" antes de tocar node.prev o node.next. Esto duplica la lógica y es fuente del bug que tuviste en LRU.

Los nodos centinela (o dummy nodes) son nodos fantasma que siempre están ahí. Tú nunca los eliminas, ellos nunca se mueven. Existen solo para que nunca tengas que checkear nil al insertar o remover.

⚠ Invariantes Sagrados
head.prev === ALWAYS nil — nada vive antes del primer centinela.
tail.next === ALWAYS nil — nada vive después del último centinela.
head y tail nunca se borran.

2 Anatomía de una Lista Vacía

Con centinelas, una lista vacía se ve así:

nil ⟵ [HEAD] ⇄ [TAIL] ⟶ nil

head apunta a tail. tail apunta a head. head.prev es nil. tail.next es nil. No hay nodos reales. La lista está vacía, pero los punteros nunca son nil entre head y tail.

3 La Soldadura de 4 Punteros

Insertar un nodo N entre dos nodos existentes A y B requiere exactamente 4 asignaciones — ni una más, ni una menos. Memoriza este patrón.

Antes:

A ⇄ B

Después de insert_after(A, val):

A ⇄ N ⇄ B
  1. A.next = N — A mira hacia N
  2. B.prev = N — B mira hacia N
  3. N.prev = A — N mira hacia atrás (A)
  4. N.next = B — N mira hacia adelante (B)
💡 El Mantra: "Primero conecto los vecinos al nuevo nodo, luego conecto el nuevo nodo a los vecinos." El orden importa — si A.next = N se hace antes de guardar B, pierdes B. Por eso en el LRU usamos las variables prev y nxt. 
def insert_after(node, val)
  n = Node.new(val)
  prev = node
  nxt  = node.next

  # Los vecinos apuntan a N
  prev.next = n
  nxt.prev  = n

  # N apunta a los vecinos
  n.prev = prev
  n.next = nxt

  n
end

4 Remove: El Puente de 2 Punteros

Remover un nodo N que está entre A y B requiere solo 2 asignaciones — conectas A directamente con B, saltándote N.

Antes:

A ⇄ N ⇄ B

Después de remove(N):

A ⇄ B (N queda huérfano — GC lo recoge) 
def remove(node)
  prev = node.prev
  nxt  = node.next

  # A y B se conectan directamente
  prev.next = nxt
  nxt.prev  = prev

  # Opcional: limpiar referencias de N
  node.prev = nil
  node.next = nil
end
⚠ ¿Puedo borrar head o tail?
No. remove(head) rompería head.prev = nil. remove(tail) rompería tail.next = nil. En tu implementación, solo llamas remove con nodos reales.

5 Operaciones Compuestas

Con insert_after y remove, todas las demás operaciones se expresan trivialmente:

OperaciónImplementación
push_front(val) insert_after(head, val)
push_back(val) insert_after(tail.prev, val) — o insert_before(tail, val)
pop_front remove(head.next)
pop_back remove(tail.prev)
empty? head.next == tail

6 Tu Error en LRU Cache (Diagnóstico)

Comparemos lo que escribiste vs. lo correcto:

Operación ❌ Tu versión ✅ Correcta
Insertar MRU prev = head.prev prev = head
Insertar MRU head.prev es nil → crash head es el centinela, insertas justo después
Evict LRU lru = tail.next lru = tail.prev
Evict LRU tail.next es nil → crash tail.prev es el último nodo real
🧠 Regla de oro: En una lista con centinelas, los nodos reales siempre viven entre head y tail. head y tail son las paredes. El MRU está pegado a head. El LRU está pegado a tail.

7 Visualización: push_front + pop_back

Sigue la secuencia paso a paso:

1. Estado inicial (vacío):

nil ⟵ [H] ⇄ [T] ⟶ nil

2. push_front(10) → insert_after(head, 10):

nil ⟵ [H] ⇄ [10] ⇄ [T] ⟶ nil

3. push_front(20):

nil ⟵ [H] ⇄ [20] ⇄ [10] ⇄ [T] ⟶ nil

4. pop_back → remove(tail.prev):

nil ⟵ [H] ⇄ [20] ⇄ [T] ⟶ nil

[10] fue removido. tail.prev ahora es [20]. La lista tiene 1 elemento real.
