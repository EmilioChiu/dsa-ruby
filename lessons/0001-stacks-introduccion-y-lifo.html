Stacks (Pilas) y la estructura LIFO

Semana 4, Día 3 • Unidad de Linked Lists, Stacks y Queues

1. El Concepto LIFO

Un Stack (Pila) es una estructura de datos lineal y dinámica regida por el principio LIFO (Last-In, First-Out): el último elemento en ser insertado es el primero en ser retirado.

Imagina una pila física de platos de cocina. Colocas cada plato limpio encima del anterior (acción de empujar o push). Cuando vas a usar un plato, tomas obligatoriamente el plato que está arriba de todo (acción de extraer o pop), ya que retirar uno del fondo causaría un desastre conceptual y físico.

Esta estructura es vital en computación. De hecho, el propio intérprete de Ruby la utiliza internamente (el Call Stack o pila de ejecución) para recordar dónde volver tras terminar de evaluar una función recursiva.

3. Implementación en Ruby

En Ruby, no necesitamos construir una estructura compleja desde cero para tener un Stack básico, ya que la clase integrada Array viene equipada con métodos altamente eficientes para este propósito.

# Inicializar una pila vacía
pila = []

# Push: Agregar elementos al tope (final del array) -> O(1)
pila.push(10)  # => [10]
pila.push(20)  # => [10, 20]
pila.push(30)  # => [10, 20, 30]

# Top / Peek: Ver el tope sin extraer -> O(1)
pila.last      # => 30

# Pop: Extraer el elemento del tope -> O(1)
elemento_extraido = pila.pop()  # => 30
# Ahora la pila queda como: [10, 20]

Complejidades de las Operaciones

Operación Complejidad Temporal Complejidad Espacial
push(x) O(1) (amortizado) O(1)
pop() O(1) O(1)
top() / peek() O(1) O(1)
Búsqueda / Acceso arbitrario O(n) O(1)
