Merge Sort & Quick Sort w6d1

Dos algoritmos de ordenamiento O(n log n) con estrategias opuestas: dividir y conquistar sin miramientos vs. elegir un lider y partir.

1. Por Que Sorting?

Los algoritmos de ordenamiento son la base de multiples problemas de entrevista:

  • Merge Intervals (w6d3) requiere ordenar por inicio
  • Top K Frequent requiere ordenar por frecuencia
  • Binary Search requiere un array ordenado
  • Two Pointers suele necesitar ordenamiento previo

Ademas, implementar Merge Sort y Quick Sort desde cero es un clasico de entrevistas para evaluar tu dominio de recursion y particionamiento.

2. Merge Sort

Merge Sort: Divide, Conquista, Combina

Tiempo: O(n log n) — siempre Espacio: O(n) — requiere array auxiliar

Estrategia:

  1. Divide: parte el array por la mitad
  2. Conquista: ordena cada mitad recursivamente
  3. Combina: fusiona las dos mitades ordenadas en una sola

La clave esta en la funcion merge: recorrer dos arrays ordenados simultaneamente, eligiendo el menor elemento en cada paso.

Visualizacion Paso a Paso

Array inicial: [38, 27, 43, 3, 9, 82, 10]
3827 433 982 10
Paso 1 — Divide:
3827 433 | 982 10

Partimos en left = [38, 27, 43, 3] y right = [9, 82, 10]

Paso 2 — Sigue dividiendo hasta tener arrays de 1 elemento:
38| 27| 43| 3| 9| 82| 10

Cada elemento solo es un array ordenado (trivialmente).

Paso 3 — Merge (combina):
2738 | 343 | 982 | 10

Fusionamos pares: (38,27) → [27,38]; (43,3) → [3,43]; (9,82) → [9,82]; (10) queda solo.

Paso 4 — Sigue fusionando:
327 3843 | 910 82

Fusionamos [27,38] + [3,43] → [3,27,38,43]; y [9,82] + [10] → [9,10,82]

Paso 5 — Fusion final:
39 1027 3843 82

Resultado: [3, 9, 10, 27, 38, 43, 82]

Implementacion en Ruby

def merge_sort(arr)
  return arr if arr.length <= 1

  mid = arr.length / 2
  left_half  = merge_sort(arr[0...mid])
  right_half = merge_sort(arr[mid..])

  merge(left_half, right_half)
end

def merge(left_half, right_half)
  result = []
  left_index = 0
  right_index = 0

  while left_index < left_half.length && right_index < right_half.length
    if left_half[left_index] <= right_half[right_index]
      result << left_half[left_index]
      left_index += 1
    else
      result << right_half[right_index]
      right_index += 1
    end
  end

  # Unir los elementos restantes (solo uno de los dos tiene sobrantes)
  result += left_half[left_index..] + right_half[right_index..]
  result
end
Detalle clave: arr[0...mid] excluye el indice mid (rango exclusivo), mientras arr[mid..] incluye desde mid hasta el final. Esto asegura que no se pierdan elementos.

3. Quick Sort

Quick Sort: Elige un Pivot, Particiona, Repite

Tiempo: O(n log n) promedio, O(n^2) peor caso Espacio: O(log n) — in-place con recursion (stack)

Estrategia:

  1. Elige un pivot (tipicamente el ultimo elemento)
  2. Particiona: mueve elementos menores que el pivot a la izquierda, mayores a la derecha
  3. Recurre: aplica Quick Sort a las dos particiones

La clave esta en la funcion partition: colocar el pivot en su posicion final tal que todo a su izquierda es menor o igual, y todo a su derecha es mayor.

Visualizacion: Particion (Lomuto)

Array: [10, 7, 8, 9, 1, 5]
107 89 15

Pivot = ultimo elemento: 5

Particion paso a paso:

i = 0 (barrera: donde van los menores)
j = 0 → 10 > 5 → no se mueve
j = 1 → 7 > 5 → no se mueve
j = 2 → 8 > 5 → no se mueve
j = 3 → 9 > 5 → no se mueve
j = 4 → 1 <= 5 → swap! arr[0]=10 ↔ arr[4]=1. i=1.

17 89 105
Colocar pivot en su lugar:

Swap arr[1]=7 ↔ pivot en posicion 5

15 89 107

Pivot 5 en posicion final. Izquierda: [1], Derecha: [8, 9, 10, 7] — se repite el proceso.

Implementacion en Ruby

def quick_sort(arr, left = 0, right = arr.length - 1)
  return arr if left >= right

  pivot_index = partition(arr, left, right)
  quick_sort(arr, left, pivot_index - 1)
  quick_sort(arr, pivot_index + 1, right)
  arr
end

def partition(arr, left, right)
  pivot_value = arr[right]
  boundary = left

  (left...right).each do |current|
    if arr[current] <= pivot_value
      arr[boundary], arr[current] = arr[current], arr[boundary]
      boundary += 1
    end
  end

  arr[boundary], arr[right] = arr[right], arr[boundary]
  boundary
end
Cuidado: Quick Sort en el peor caso (array ya ordenado con pivot = ultimo) es O(n^2). Para mitigarlo, se usa un pivot aleatorio o la estrategia de "mediana de tres".

4. Comparacion Directa

Propiedad Merge Sort Quick Sort
Tiempo promedio O(n log n) O(n log n)
Peor caso O(n log n) — siempre igual O(n^2) — array ordenado + mal pivot
Espacio extra O(n) — array auxiliar O(log n) — in-place (stack recursion)
Estable? Si — preserva orden relativo No — puede reordenar elementos iguales
In-place? No Si
Uso tipico Cuando la estabilidad importa o peor caso no es aceptable Cuando el espacio es limitado
