Binary Search O(log n)

De buscar en un diccionario a encontrar elementos en arrays ordenados. El primer patron que realmente aprovecha la estructura de los datos.

1. La Intuicion

Buscas "manzana" en un diccionario de 1000 paginas. Abres por la mitad (~p. 500). Ves que estas en la "M". "Manzana" esta antes. Descartas 500 paginas de un golpe. Repites: abres entre p. 1 y p. 500. Esa es la esencia de Binary Search.

En cada paso, descartas la mitad del espacio de busqueda. En un array de 1,000,000 elementos, la busqueda lineal toma hasta 1,000,000 pasos. Binary Search toma a lo sumo 20 pasos (2^20 ~ 1,048,576).

Requisito absoluto: el array debe estar ordenado. Sin orden, no hay base para descartar mitades.

2. Visualizacion Paso a Paso

Buscamos target = 23 en nums = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]:

Paso 1: left=0, right=9, mid=4 → nums[4]=16 < 23 → descartamos izquierda

Paso 2: left=5, right=9, mid=7 → nums[7]=56 > 23 → descartamos derecha
L 23 38 56 72 R → L 23 R
Paso 3: left=5, right=6, mid=5 → nums[5]=23 == target → encontrado!

3. La Plantilla Estandar

Esta es la unica plantilla que necesitas. Memorizala.

def binary_search(nums, target)
  left  = 0
  right = nums.length - 1

  while left <= right
    mid = left + (right - left) / 2

    if nums[mid] == target
      return mid
    elsif nums[mid] < target
      left  = mid + 1
    else
      right = mid - 1
    end
  end

  return -1
end
Por que left + (right - left) / 2? En C++/Java, (left + right) / 2 puede desbordar si los indices son enormes. En Ruby los enteros no desbordan, pero usar esta forma muestra que conoces el detalle.

Los 3 Puntos Criticos

Elemento Que hace Error comun
while left <= right Incluye el ultimo elemento Usar < y saltarte el ultimo
mid = left + (right - left) / 2 Calcula el punto medio Olvidar recalcularlo en cada iteracion
left = mid + 1 / right = mid - 1 Descarta la mitad equivocada Usar left = mid (loop infinito)

4. Variantes Clasicas

La plantilla se adapta ligeramente segun lo que pida el problema:

Problema Cambio Retorno
LeetCode 704 Plantilla exacta mid si existe, -1 si no
LeetCode 35 Si no existe, retorna left mid o left
LeetCode 278 En vez de nums[mid], llamas a isBadVersion(mid) left (primer bad version)

5. Ponte a Prueba

Pregunta 1

Si un array tiene 128 elementos, cual es el maximo numero de comparaciones que hara Binary Search?

Ver respuesta
7. Porque 2^7 = 128. Cada comparacion reduce el espacio a la mitad.

Pregunta 2

Que pasa si usas while left < right en vez de <= en un array de 1 elemento donde ese unico elemento es el target?

Ver respuesta
No lo encuentra. Con left=0, right=0, la condicion 0 < 0 es falsa de entrada, asi que el loop nunca se ejecuta y retorna -1.

Pregunta 3

Olvidas los +1 y -1 al actualizar los limites: usas left = mid y right = mid. Que ocurre?

Ver respuesta
Loop infinito. Si left = 3, right = 4, mid = 3, y nums[3] < target, entonces left = 3 otra vez. Nunca avanzas.
