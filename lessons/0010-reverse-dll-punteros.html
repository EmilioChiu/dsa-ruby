0010: Punteros al Revés

Reverse Doubly Linked List — swapping prev/next

Lección Semana 4 — Deuda Técnica Prerequisito: Lección 0009

1 ¿Qué significa "reversar" una DLL?

Invertir una Doubly Linked List significa que el head actual se convierte en tail, y el tail actual se convierte en head. Para cada nodo, los punteros prev y next se intercambian.

Antes:

1 ⇄ 2 ⇄ 3 ⇄ 4
head → 1, tail → 4

Después:

4 ⇄ 3 ⇄ 2 ⇄ 1
head → 4, tail → 1
⚠ Sin centinelas
Este ejercicio se hace con nodos puros (sin head/tail dummy). El primer nodo tiene prev = nil, el último tiene next = nil. Después del reverse, el que era primero ahora es último.

2 El Algoritmo: Swap Universal

La idea es simple: recorres cada nodo y swapeas su prev con su next. Al final, retornas el nodo que solía ser el tail (el nuevo head). 

def reverse_dll(head)
  current = head
  prev_node = nil

  while current
    # Guardar referencia al next ANTES de romperla
    nxt = current.next

    # Swap: el next apunta a lo que era prev, y viceversa
    current.next = current.prev
    current.prev = nxt

    # Avanzar al siguiente nodo (que era el "prev" original... o espera, el next?)
    prev_node = current
    current = nxt
  end

  # prev_node es el último nodo procesado = nuevo head
  prev_node
end
🧠 Clave: Antes de swap, guarda nxt = current.next. Después del swap, current.next ya apunta al viejo current.prev. Para avanzar al siguiente nodo de la lista original, usa el nxt que guardaste.

3 Paso a Paso Visual

Iteración 1: current = 1 (head)

Estado antes del swap
current = 1  |  prev = nil  |  next = 2
prev_node = nil
Después del swap
1.next = 2 → 1.next = nil
1.prev = nil → 1.prev = 2
prev_node = 1, current = 2

Iteración 2: current = 2

Antes del swap
current = 2  |  prev = 1  |  next = 3
Después del swap
2.next = 3 → 2.next = 1
2.prev = 1 → 2.prev = 3
prev_node = 2, current = 3

Iteración 3: current = 3

Antes del swap
current = 3  |  prev = 2  |  next = 4
Después del swap
3.next = 4 → 3.next = 2
3.prev = 2 → 3.prev = 4
prev_node = 3, current = 4

Iteración 4: current = 4 (último)

Antes del swap
current = 4  |  prev = 3  |  next = nil
Después del swap
4.next = nil → 4.next = 3
4.prev = 3 → 4.prev = nil
prev_node = 4, current = nil → fin del loop

Resultado: prev_node = 4. Retornamos 4.

4 ⇄ 3 ⇄ 2 ⇄ 1
head → 4 (prev = nil), tail → 1 (next = nil)

4 Edge Cases

CasoQué esperar
head == nil Retornar nil
head.next == nil (1 nodo) Retornar el mismo nodo (sus punteros no cambian)
2 nodos Simplemente se intercambian. head = 1, tail = 2 → head = 2, tail = 1
N lista grande El último nodo procesado es el nuevo head
⚠ Error común: Al final del loop, retornar current en lugar de prev_node. current siempre es nil cuando el loop termina. prev_node es el último nodo procesado = nuevo head.

5 Visualización Alternativa

Otra forma de pensarlo: imagina que cada nodo es una persona en una fila. "Reverse" significa que todos dan media vuelta. Tu prev se convierte en next y viceversa. El que estaba al frente ahora está atrás.

Antes:
nil ⟵ 1 ⟷ 2 ⟷ 3 ⟷ 4 ⟶ nil
head tail

Después:
nil ⟵ 4 ⟷ 3 ⟷ 2 ⟷ 1 ⟶ nil
head tail

6 Conectando con el LRU Cache

¿Por qué esto importa para el LRU? Porque en el LRU mueves nodos constantemente: cuando haces get(key), desconectas el nodo de su posición actual (similar a removerlo de la lista) y lo reconectas al frente.

Reverse DLL te fuerza a pensar en las direcciones de los punteros sin el "andamio" de los centinelas. Si puedes hacer reverse DLL sin errores, las 4 conexiones del insert_after te parecerán triviales.
