Lección 0002: El Patrón de Monotonic Stack

Has identificado correctamente el problema: las soluciones de fuerza bruta nos llevan a un tiempo O(n²). En entrevistas, si ves "siguiente elemento más grande", tu cerebro debe gritar: ¡Monotonic Stack!

La Intuición: "La Sala de Espera"

Regla de Oro: El Stack siempre guardará elementos en orden descendente. En cuanto llega una temperatura más alta que la del tope, ¡esa temperatura "libera" (pops) a los que están esperando!

Simulación Paso a Paso

Temperaturas: [73, 74, 75, 71, 69, 72]

Inicio - Inicializar ans con 0s [] [0, 0, 0, 0, 0, 0]
0 73 Push 0 [0] [0, 0, 0, 0, 0, 0]
1 74 74 > 73? SÍ. Pop 0.
ans[0] = 1 - 0 = 1.
Push 1		 [1] [1, 0, 0, 0, 0, 0]
2 75 75 > 74? SÍ. Pop 1.
ans[1] = 2 - 1 = 1.
Push 2		 [2] [1, 1, 0, 0, 0, 0]
3 71 71 > 75? NO.
Push 3		 [2, 3] [1, 1, 0, 0, 0, 0]
4 69 69 > 71? NO.
Push 4		 [2, 3, 4] [1, 1, 0, 0, 0, 0]
5 72 72 > 69? SÍ. Pop 4. ans[4]=5-4=1
72 > 71? SÍ. Pop 3. ans[3]=5-3=2
72 > 75? NO.
Push 5 		[2, 5] [1, 1, 0, 2, 1, 0]
Observación: Los índices que quedan en el stack (2 y 5) se quedan con 0 en el resultado porque nunca encontraron un día más caliente.

Pseudocódigo para Ruby


def daily_temperatures(temperatures)
  ans = Array.new(temperatures.length, 0)
  stack = [] # Guardaremos ÍNDICES

  temperatures.each_with_index do |t, i|
    # Mientras haya alguien esperando Y la temp actual sea mayor...
    while !stack.empty? && t > temperatures[stack.last]
      prev_i = stack.pop
      ans[prev_i] = i - prev_i
    end
    stack << i
  end

  ans
end
