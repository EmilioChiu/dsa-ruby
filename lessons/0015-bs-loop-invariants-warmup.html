Warmup: BS Loop Invariants debt-002

Practica eligiendo entre left < right vs left <= right, y cuando usar mid, mid - 1, mid + 1.

Deuda conceptual debt-002: Confusion entre left < right vs left <= right, y como afecta el manejo de boundaries. Resuelve los 8 escenarios para cerrar esta deuda.

Referencia Rapida

Los 4 Movimientos de BS

MovimientoSignificaCondicion
left = mid + 1 mid y todo a la izq no pueden ser respuesta Sabes que mid no es valido
right = mid - 1 mid y todo a la der no pueden ser respuesta Sabes que mid no es valido
left = mid mid podria ser la respuesta (conservalo) No puedes descartar mid
right = mid mid podria ser la respuesta (conservalo) No puedes descartar mid

Cuando usar cual while?

CondicionRangoMovimiento seguroEjemplo tipico
while left <= right [left, right] con 1 elemento left = mid + 1, right = mid - 1 BS clasico (LC 704)
while left < right [left, right) — rango exclusivo left = mid + 1, right = mid BS en respuesta (LC 875)

Resumen: Arbol de Decision

Regla de oro:
left <= right → siempre puedes mover mid +/- 1 (ya lo descartaste).
left < right → si conservas mid (= mid), usa mid = ceil() (mid superior) para evitar loop infinito.
Tip: Si no estas seguro, preguntate:
"Puedo probar que mid NO es la respuesta?"
Si si → mid +/- 1 (descartalo).
Si no → = mid (conservalo como candidato).
