BS en Respuesta Analisis

Ya sabes buscar en un array ordenado. Ahora aprende a buscar en el espacio de respuestas posibles — un patron que aparece cuando el problema pregunta por un minimo o maximo que cumple una condicion.

1. BS Clasico vs. BS en Respuesta

En la leccion 0006 aprendiste el BS clasico: buscas un target dentro de un array ordenado. El array es tu espacio de busqueda.

BS Clasico BS en Respuesta
Que buscas? Un valor en un array El numero minimo/maximo que cumple X
Espacio de busqueda nums[0..n-1] [min..max] — rango numerico
Condicion nums[mid] == target feasible?(mid) == true
Monotonicidad Array ordenado asc/desc La condicion es monotona: F..F..T..T o T..T..F..F
Template while left <= right while left < right

2. La Intuicion: "Cual es el minimo X tal que...?"

Muchos problemas de entrevista preguntan por un minimo o un maximo:

  • "Cual es la velocidad minima para comer todos los bananas en H horas?" (Koko)
  • "Cual es la capacidad minima del barco para enviar paquetes en D dias?" (Ship Packages)
  • "Cual es la raiz cuadrada de X?" (Sqrt)

En todos estos casos, el array NO es el espacio de busqueda — el rango de valores posibles de la respuesta lo es.

La clave: Si puedes escribir una funcion feasible?(candidato) que retorna true/false y esa funcion es monotónica (nunca pasa de true a false, o viceversa), entonces puedes aplicar BS sobre el rango de candidatos.

3. La Plantilla Generica

def find_min_feasible(problem)
  left  = MINIMO_VALIDO      # el menor valor que podria funcionar
  right = MAXIMO_POSIBLE     # el mayor valor que podria funcionar

  while left < right          # estrechamos hasta encontrar el limite
    mid = left + (right - left) / 2

    if feasible?(mid)         # mid funciona? busca uno menor
      right = mid
    else                       # mid no funciona? necesita uno mayor
      left  = mid + 1
    end
  end

  left                       # = right, el minimo valor factible
end
Cuidado: Aqui usamos while left < right, NO <=. Por que? Porque no estamos buscando un valor exacto — estamos estrechando un rango hasta que left == right, que es el punto exacto donde la condicion cambia de false a true. Si usas <=, puedes caer en un loop infinito.

Cuando el maximo es el que buscas

A veces el problema pide el maximo valor que aun cumple algo. En ese caso, invierte la logica:

while left < right
  mid = left + (right - left + 1) / 2  # mid "alto" para evitar loop infinito

  if feasible?(mid)
    left  = mid               # mid funciona, busca uno mayor
  else
    right = mid - 1           # mid no funciona, busca uno menor
  end
end

4. Ejemplo Guiado: Sqrt(x)

Buscamos la raiz cuadrada de x = 17, redondeada hacia abajo.

Paso 1: Identificar el rango

La raiz de 17 esta entre 0 y 17. Mas precisamente, entre 1 y 17. left = 1, right = 17.

Paso 2: Definir feasible?

feasible?(k) = k * k <= x

Por que? Si k * k <= 17, entonces k podria ser la raiz (o menor). Buscamos el maximo k que cumple esto.

Patron: Nota que k * k <= x es monotónica: [1, 2, 3, 4] cumplen, [5, 6, 7, ..., 17] no cumplen. False, False, ..., True, True.
Busqueda en rango [1..17] para x=17:
L=1 2 3 4 mid=9 10 11 12 13 14 15 16 R=17

mid=9, 9*9=81 > 17 — no factible. right = mid - 1 = 8

L=1 2 3 mid=4 5 6 7 R=8

mid=4, 4*4=16 <= 17 — factible. left = mid = 4

L=4 5 mid=6 7 R=8

mid=6, 6*6=36 > 17 — no factible. right = mid - 1 = 5

L=4 mid=5 R=5

mid=5, 5*5=25 > 17 — no factible. right = mid - 1 = 4

L=4 R=4

left == right == 4. Retornamos 4. Correcto!

5. Ejemplo Guiado: Koko Eating Bananas

piles = [3, 6, 7, 11], h = 8. Buscamos la velocidad minima k para comer todos.

Identificar el rango

La velocidad minima posible es 1 (una banana por hora). La maxima es max(piles) = 11 (la pila mas grande). left = 1, right = 11.

Definir feasible?

Para una velocidad k, calculamos cuantas horas toma comer todas las pilas:

def can_eat_all?(piles, h, k)
  hours = piles.sum do |pile|
    (pile + k - 1) / k  # ceiling division
  end
  hours <= h
end

Esta funcion es monotónica: si k=4 funciona, k=5,6,...11 tambien funcionan.

Ceiling division: (pile + k - 1) / k es equivalente a ceil(pile / k.to_f) pero sin punto flotante. Memoriza este idiom.

Con k=4: hours = ceil(3/4)+ceil(6/4)+ceil(7/4)+ceil(11/4) = 1+2+2+3 = 8 <= 8. Funciona!

Con k=3: hours = 1+2+3+4 = 10. No funciona.

Como feasible? es F-F-F-T-T-T, buscamos el primer true. Respuesta: 4.

6. El Patron para Identificarlos

Cuando veas un problema, hazte 3 preguntas:

  1. Pide un minimo o maximo? "Minimum k such that...", "Least capacity..."
  2. Puedo escribir feasible?(candidato)? Una funcion que simule si ese candidato funciona
  3. Es monotónica? Si X funciona, todo Y > X tambien funciona (o viceversa)

Si respondes si a las 3, es BS en respuesta.

7. Ponte a Prueba

Pregunta 1

En Koko, el rango de busqueda es [1..max(piles)]. Por que max(piles) y no sum(piles)?

Ver respuesta
Porque si k = max(piles), Koko puede comerse la pila mas grande en 1 hora. Ir mas rapido no reduce las horas (ya no puede comer mas de una pila por hora). max(piles) es el limite superior natural — mas alla no cambia el resultado.

Pregunta 2

En Ship Packages, cual es el rango de busqueda para la capacidad del barco?

Ver respuesta
left = max(weights) (el paquete mas pesado debe caber) y right = sum(weights) (si enviaras todo en un solo dia).

Pregunta 3

Que pasa si feasible? no es monotónica? Por ejemplo, funciona con k=5, no con k=6, si con k=7?

Ver respuesta
BS no funciona. El algoritmo depende de poder descartar mitades del rango. Sin monotonicidad, no hay base para decidir que mitad descartar. Tendrias que usar busqueda lineal o fuerza bruta.
