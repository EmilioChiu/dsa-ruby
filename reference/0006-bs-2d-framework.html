Mapeo 2D ↔ 1D

Para una matriz m x n que se puede aplanar en un array ordenado:

row = mid / n
col = mid % n
val = matrix[row][col]

BS Clasico en 2D (LC 74)

def search_matrix(matrix, target)
  m = matrix.length
  n = matrix[0].length
  left = 0
  right = m * n - 1

  while left <= right
    mid = left + (right - left) / 2
    val = matrix[mid / n][mid % n]

    if val == target
      return true
    elsif val < target
      left = mid + 1
    else
      right = mid - 1
    end
  end
  false
end
Requisito: La matriz debe estar "fully sorted" — cada fila ordenada Y primer elemento de fila i+1 > ultimo de fila i. Si no, no puedes aplanarla.

Los 4 Patrones de BS

Patron Señal mid se descarta? Problemas
Clasico 1 Array ordenado, target exacto Si (ya se comparo) LC 704, 35, 278
En respuesta 2 Rango de valores, feasible? monotona No (podria ser el minimo factible) LC 875, 1011, 69
Rotado 3 Array ordenado + pivot Depende LC 33, 153, 162
2D 4 Matriz m×n fully sorted Si (igual al clasico) LC 74

Arbol de Decision Rapido

1. Array plano ordenado → BS clasico

target conocido, buscas existencia.

2. Sin array, rango de valores → BS en respuesta

Pregunta: "cual es el minimo X tal que Y?"

3. Array ordenado pero rotado → BS rotado

Buscar pivot o target en array con quiebre.

4. Matriz m×n fully sorted → BS 2D

Aplana virtualmente con row = mid / n.

X. Ninguna monotonia clara → No es BS

Busca otro patron: hash, pointers, sliding window, etc.

Off-by-One: Recordatorio

  • mid podria ser la respuesta? → right = mid (o left = mid con ceiling)
  • mid NO puede ser la respuesta? → mid +/- 1
  • left = mid necesita ceiling: (left + right + 1) / 2
  • right = mid es siempre seguro
