Stack (Pila)

Estructura LIFO — último en entrar, primero en salir

★ Operaciones

En Ruby, un Array funciona como stack nativo: push / << agregan al final, pop quita del final.

⚙ Patrones Clave

  • Parsing / Validacion: emparejar simbolos, parentesis, brackets. Empujar opens, validar con pop.
  • Two-Stack: stack principal + stack auxiliar (ej. Min Stack — guardar minimos en paralelo).
  • Monotonic Stack: mantener stack ordenado (creciente/decreciente). Next Greater Element, Daily Temperatures.
  • Evaluacion de expresiones: RPN, shunting-yard. Operandos al stack, operadores desempacan.
  • Undo / Backtracking: historial de estados (navegador, editor de texto).

🛠 Implementacion desde Cero

class Stack
  def initialize
    @data = []
  end

  def push(val)
    @data << val
  end

  def pop
    @data.pop
  end

  def peek
    @data.last
  end

  def empty?
    @data.empty?
  end

  def size
    @data.length
  end
end

📈 Monotonic Stack (Decreciente)

def next_greater(arr)
  stack = []
  result = Array.new(arr.size, -1)

  arr.each_with_index do |val, i|
    while !stack.empty? && val > arr[stack.last]
      idx = stack.pop
      result[idx] = val
    end
    stack << i
  end

  result
end

El stack guarda indices, no valores. Asi puedes asignar al resultado en la posicion correcta.

📄 Consejos de Entrevista

  • Menciona LIFO apenas veas un problema de anidamiento o "orden inverso"
  • Stack con indices (no valores) cuando necesites posiciones en el resultado
  • Si necesitas minimo/maximo O(1), piensa en two-stack
  • Array en Ruby push/pop son nativos y eficientes — no necesitas implementar nada
  • Pregunta si los valores pueden repetirse (afecta logica de Min Stack)
