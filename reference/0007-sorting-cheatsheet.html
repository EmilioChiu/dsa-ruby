Sorting Cheatsheet

Merge Sort vs Quick Sort — implementacion, complejidad, decision

Merge Sort

def merge_sort(arr)
  return arr if arr.length <= 1
  mid = arr.length / 2
  left_half  = merge_sort(arr[0...mid])
  right_half = merge_sort(arr[mid..])
  merge(left_half, right_half)
end

def merge(left_half, right_half)
  result = []
  left_index = 0; right_index = 0
  while left_index < left_half.length && right_index < right_half.length
    if left_half[left_index] <= right_half[right_index]
      result << left_half[left_index]; left_index += 1
    else
      result << right_half[right_index]; right_index += 1
    end
  end
  result += left_half[left_index..] + right_half[right_index..]
  result
end

Quick Sort (Lomuto)

def quick_sort(arr, left = 0,
               right = arr.length - 1)
  return arr if left >= right
  pivot_index = partition(arr, left, right)
  quick_sort(arr, left, pivot_index - 1)
  quick_sort(arr, pivot_index + 1, right)
  arr
end

def partition(arr, left, right)
  pivot_value = arr[right]
  boundary = left
  (left...right).each do |current|
    if arr[current] <= pivot_value
      arr[boundary], arr[current] = arr[current], arr[boundary]
      boundary += 1
    end
  end
  arr[boundary], arr[right] = arr[right], arr[boundary]
  boundary
end

A Favor

  • Rendimiento garantizado O(n log n)
  • Estable (preserva orden de elementos iguales)
  • Predecible: sin variacion por datos de entrada

A Favor

  • In-place: O(1) espacio extra en particion
  • Cache-friendly: acceso secuencial
  • En la practica, suele ser mas rapido que Merge Sort

En Contra

  • Requiere O(n) memoria extra
  • Mas lento en la practica que Quick Sort (allocations)

En Contra

  • Peor caso O(n^2) con pivot ingenuo
  • No estable
  • Stack overflow posible en arrays grandes

Mejoras a Quick Sort

Pivot aleatorio: pivot = arr[rand(lo..hi)] — evita el peor caso en la practica.

Mediana de tres: elegir pivot como la mediana de arr[lo], arr[mid], arr[hi].

Hoare partition: alternativa a Lomuto que hace ~3x menos swaps (usada en implementaciones reales).

Arbol de Decision

Usa Merge Sort cuando:

  • La estabilidad es critica (ej: ordenar por fecha manteniendo prioridad)
  • El peor caso debe ser garantizado O(n log n)
  • La memoria no es una restriccion (arrays pequenos o mucha RAM)

Usa Quick Sort cuando:

  • La memoria es limitada (embedded, sistemas con restricciones)
  • Rendimiento bruto importa mas que estabilidad
  • Los datos son aleatorios o puedes randomizar el pivot
